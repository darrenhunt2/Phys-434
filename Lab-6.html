Finding the $5 \sigma$ Sensitivity Threshold¶

To find the 5-sigma threshold for our background distribution, we first convert 5 sigma for a Gaussian into its corresponding probability (using cumulative distribution function), then take the inverse with the Poisson's percent point function. This answer will tell us the value 5 standard deviations from the mean which is the cutoff point for a statistically significant observation (above 5 sigma). Anything below this cutoff, we are not guaranteed it is not just part of the background.

In [4]:
# calculate 5sigma threshold
p = stats.norm.cdf(5)
thresh = stats.poisson.ppf(p,mu=100)
print("Sensitivity threshold for background = "+str(thresh))

Sensitivity threshold for background = 154.0

The cutoff value is 154. Any number of events lower than this is not statisticially significant and may be part of the background distribution.

These histograms show us the background and the observed data distributions post-signal injection. The log plot shows more clearly the true distribution shapes and the minor changes along the curves; as we can see, both the background and the observation shapes appear parabolic on the log plot and therefore approached a Gaussian distribution.

After signal injection into the background, there are two notable changes to the histogram distributions: One, the peaks have changed. Both curves have shorter peaks (in counts) than pre-signal injection. Second, most importantly, the observed signal has changed. While the background remains the same as expected, since our fake background doesn't change with time or any other expected changes, the signal histogram shape has now shifted up on the x-axis. Looking closer, we see the peak, where the mean is located, has shifted up exactly by 100, from 215 to 315.

This upwards shift along the x-axis happens due to the additive nature of the background. The background mean is 100, therefore all values within the signal distribution have 100 added.

What we are looking at then is the background and the signal together. The true signal is not actually this much stronger. Instead, the background is just adding to the value we observe or measure. To get the true signal, we would have to subtract the background out in a real experiment.

The observed signal above is very symmetric about the mean=215 signal strength, as seen in both the histogram and log plots above. There is very little variation or change away from a symmetric Gaussian shape. Since both the background and the signal follow the same type of distribution, we don't expect the observed signal value to stray far from the Gaussian. There is however a bias in the observed signal. This is because the observed signal is not equally probable to be stronger or weaker than the true injected signal. The background can only add positive values here, and the range of injected signals is relatively small. The observed signal resulting from this background therefore will never be below what the true value is. We can expect this sort of bias due to the nature of the signal we injected. It took on a Poisson distribution about a pre-defined mean, instead of random and uniform signals across a very wide range.

Injecting a Random, Uniform Range of Signals¶

Instead of injecting a signal Poisson-distributed about a single mean, this time we can randomly generate a large amount of signals in a given range. We can start at 0, and going several sigmas above the 5-sigma threshold of 154 events, we can stop a few hundred events greater, say 400. In order to avoid an implicit prior, we need to ensure the random values are uniformly distributed, and have equal probabilities of strength at either end of the signal range.

These histograms now show us the possible true signal values if we observe a value of 353. The true signal holds its Gaussian shape according to the log plot. Looking along the x-axis, we can see that the true signal values possible are 100 less than the observed. This is because the background is subtracted out for a true signal, and as before, the background mean is 100. Therefore, a true signal significantly lower than the observed signal is what we expect.

There is still a range of possible values for the inverted problem. Though we have measured a value of 353, the signal still has a spread. Again, this is due to uncertainty from the background. The background may add a little more or less than the mean. We still expect the mean to be most probable, but the true signal could fall on either side of this mean according to the background.

Since there is this spread in the possible true signal values, we can calculate the 1$\sigma$ uncertainty. The 1$\sigma$ uncertainty tells us that 67% of the time, the true signal will fall within one sigma from the mean on either side.

In [24]:
# calculate 1 sigma uncertainty
# calculate each side separately to ensure it's symmetric
onesig = stats.norm.cdf(1)
onesigneg = stats.norm.cdf(-1)
upper = stats.poisson.ppf(onesig,mu=tmean)
lower = stats.poisson.ppf(onesigneg,mu=tmean)

plus = upper - tmean 
minus = tmean - lower

print("1-sigma uncertainty, upper : "+str("{0:.2f}".format(plus))+", Upper bound : "+str(upper))
print("1-sigma uncertainty, lower : "+str("{0:.2f}".format(minus))+", Lower Bound : "+str(lower))

1-sigma uncertainty, upper : 15.58, Upper bound : 269.0
1-sigma uncertainty, lower : 15.42, Lower Bound : 238.0

Therefore, we can expect a true signal of the mean, plus or minus these bounds, within the 1-sigma uncertainty confidence interval. We are pretty confident the true signal falls somewhere within that range.

I calculated both the upper and lower bounds separately to check if it truly is symmetric. The above calculations show that it is symmetric about the mean. We can expect the values to evenly fall within 1 sigma of either side, a value of plus or minus 15 events. The symmetry shows that you are equally likely to fall left of the mean as you are to fall to the right. Therefore, we can most likely expect the mean, or another value within + or - 15 of the mean.

This makes sense. Our data is randomly generated each time this code is rerun, but it is expected to follow a Poisson distribution regardless. With many samples, we see the shape approaching to a Gaussian distribution, symmetric about whichever mean we input. In this lab, we used a mean of 100 for the background. Throughout the lab, I only saw minor fluctuations from the expected shape, but it never deviated from an approximately symmetric shape. Additionally, we made sure to randomly generate the signal as uniform to avoid weighting either side too heavily. If we had not generated this uniformly, we would likely see some asymmetry or more bias. The uniform range of signals is an attempt to eliminate the bias that we saw in the previous problem.
